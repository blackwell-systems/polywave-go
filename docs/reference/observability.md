# Observability ‚Äî Reference

Package `pkg/observability` provides event types, a storage abstraction, and query
helpers for tracking cost, agent performance, and orchestrator activity across SAW
executions.

## Architecture

```
Orchestrator / Emitter
        ‚îÇ
        ‚ñº RecordEvent
  Store interface
        ‚îÇ
  ‚îå‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚î¥‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îê
  ‚ñº            ‚ñº
SQLite     (PostgreSQL ‚Äî not yet implemented)
~/.polywave/observability.db
```

The `Store` interface is the only dependency for write and read paths. The SQLite
implementation (`pkg/observability/sqlite`) is the current default. PostgreSQL DSNs
are accepted by the CLI but return an error at runtime.

The `Emitter` struct wraps `Store` with a nil-safe, fire-and-forget API. A nil
`*Emitter` is safe to call on every method ‚Äî all calls become no-ops.

## Event types

Three concrete types implement the `Event` interface. The `type` field in each
record determines how it is stored and deserialized.

### Event interface

| Method | Return | Description |
|---|---|---|
| `EventID()` | `string` | UUID v4 identifier, generated by `NewEventID()` |
| `EventType()` | `string` | `"cost"`, `"agent_performance"`, or `"activity"` |
| `Timestamp()` | `time.Time` | UTC timestamp of the event |
| `Metadata()` | `map[string]any` | Arbitrary extra fields; may be nil |

### CostEvent

Emitted once per agent invocation to record token usage and cost.

| Field | Type | JSON key | Description |
|---|---|---|---|
| ID | string | `id` | UUID v4 |
| Type | string | `type` | `"cost"` |
| Time | time.Time | `time` | UTC timestamp |
| AgentID | string | `agent_id` | Agent identifier (e.g. `"A"`, `"B"`) |
| WaveNumber | int | `wave_number` | Wave this agent ran in |
| IMPLSlug | string | `impl_slug` | IMPL feature slug |
| ProgramSlug | string | `program_slug` | Program slug (omitted if not in a program) |
| Model | string | `model` | Model name (e.g. `"claude-sonnet-4-6"`) |
| InputTokens | int | `input_tokens` | Tokens in the prompt |
| OutputTokens | int | `output_tokens` | Tokens in the response |
| CostUSD | float64 | `cost_usd` | Estimated cost in USD |
| Meta | map[string]any | `metadata` | Arbitrary extra data |

### AgentPerformanceEvent

Emitted once per agent run to record execution outcome, duration, and test results.

| Field | Type | JSON key | Description |
|---|---|---|---|
| ID | string | `id` | UUID v4 |
| Type | string | `type` | `"agent_performance"` |
| Time | time.Time | `time` | UTC timestamp |
| AgentID | string | `agent_id` | Agent identifier |
| WaveNumber | int | `wave_number` | Wave number |
| IMPLSlug | string | `impl_slug` | IMPL feature slug |
| ProgramSlug | string | `program_slug` | Program slug (omitted if absent) |
| Status | string | `status` | `"success"`, `"failed"`, `"blocked"`, or `"partial"` |
| FailureType | string | `failure_type` | `"transient"`, `"fixable"`, `"needs_replan"`, `"escalate"`, or `"timeout"` (empty on success) |
| RetryCount | int | `retry_count` | Number of retries before this outcome |
| DurationSeconds | int | `duration_seconds` | Wall-clock execution time |
| FilesModified | []string | `files_modified` | Paths touched by the agent |
| TestsPassed | int | `tests_passed` | Count of passing tests |
| TestsFailed | int | `tests_failed` | Count of failing tests |
| Meta | map[string]any | `metadata` | Arbitrary extra data |

### ActivityEvent

Emitted for high-level orchestrator lifecycle transitions. Multiple `activity_type`
values share this type.

| Field | Type | JSON key | Description |
|---|---|---|---|
| ID | string | `id` | UUID v4 |
| Type | string | `type` | `"activity"` |
| Time | time.Time | `time` | UTC timestamp |
| ActivityType | string | `activity_type` | See table below |
| IMPLSlug | string | `impl_slug` | IMPL feature slug |
| ProgramSlug | string | `program_slug` | Program slug (omitted if absent) |
| WaveNumber | int | `wave_number` | Wave number (0 for non-wave events) |
| User | string | `user` | User who triggered the action (optional) |
| Details | string | `details` | Free-form detail string (optional) |
| Meta | map[string]any | `metadata` | Arbitrary extra data |

#### ActivityType values

| Value | Emitter constructor | When emitted |
|---|---|---|
| `scout_launch` | `NewScoutLaunchEvent` | Scout agent starts |
| `scout_complete` | `NewScoutCompleteEvent` | Scout agent finishes successfully |
| `wave_start` | `NewWaveStartEvent` | Wave begins execution |
| `wave_merge` | `NewWaveMergeEvent` | Wave merge completes (counted as wave_completed in metrics) |
| `wave_failed` | `NewWaveFailedEvent` | Wave merge fails; `Details` contains reason |
| `impl_complete` | `NewImplCompleteEvent` | All waves merged, IMPL is done |
| `gate_executed` | `NewGateExecutedEvent` | Quality gate passed; `Details`: `gate=<type> passed=true` |
| `gate_failed` | `NewGateExecutedEvent` | Quality gate failed; `Details`: `gate=<type> passed=false` |
| `tier_advanced` | `NewTierAdvancedEvent` | Program tier advances; `WaveNumber` = tier number |
| `tier_gate_passed` | `NewTierGatePassedEvent` | Program tier gate passes |
| `tier_gate_failed` | `NewTierGateFailedEvent` | Program tier gate fails; `Details` contains reason |

## Store interface

```go
type Store interface {
    RecordEvent(ctx context.Context, event Event) error
    QueryEvents(ctx context.Context, filters QueryFilters) ([]Event, error)
    GetRollup(ctx context.Context, rollup RollupRequest) (*RollupResult, error)
    Close() error
}
```

The SQLite store (`pkg/observability/sqlite`) is the production implementation.
Database file defaults to `~/.polywave/observability.db`. WAL mode is enabled at open
time. Schema is applied automatically on first open.

### SQLite schema

```sql
CREATE TABLE IF NOT EXISTS events (
    id   TEXT PRIMARY KEY,
    type TEXT NOT NULL,
    time TEXT NOT NULL,   -- RFC3339Nano, UTC
    data TEXT NOT NULL    -- full JSON of the event
);
CREATE INDEX IF NOT EXISTS idx_events_type ON events(type);
CREATE INDEX IF NOT EXISTS idx_events_time ON events(time);
```

Filters on `impl_slug`, `program_slug`, and `agent_id` are applied in Go after the
SQL query (those fields live inside the JSON `data` column).

## QueryFilters

| Field | Type | Description |
|---|---|---|
| EventTypes | []string | Filter to one or more event types. Empty = all types. |
| IMPLSlugs | []string | Filter to specific IMPL slugs. |
| ProgramSlugs | []string | Filter to specific program slugs. |
| AgentIDs | []string | Filter to specific agent IDs. |
| StartTime | *time.Time | Inclusive start of time range. |
| EndTime | *time.Time | Inclusive end of time range. |
| Limit | int | Maximum rows returned. 0 = no limit (used by aggregation functions). |
| Offset | int | Row offset for pagination. Only applied when Limit > 0. |

Results are ordered by `time ASC`.

## Rollups and aggregations

### RollupRequest

| Field | Type | Description |
|---|---|---|
| Type | string | `"cost"`, `"success_rate"`, or `"retry_count"` |
| GroupBy | []string | Dimensions: `"agent"`, `"wave"`, `"impl"`, `"program"`, `"model"` |
| IMPLSlug | string | Restrict to one IMPL. |
| ProgramSlug | string | Restrict to one program. |
| StartTime | *time.Time | Start of time range. |
| EndTime | *time.Time | End of time range. |

### RollupResult

| Field | Type | Description |
|---|---|---|
| Type | string | Mirrors `RollupRequest.Type` |
| Groups | []RollupGroup | One entry per group-by bucket |
| TotalCost | float64 | Sum across all groups (cost rollup only) |
| AvgRate | float64 | Average cost per group (cost) or average success rate across groups (success_rate) |

### RollupGroup

| Field | Type | Description |
|---|---|---|
| Key | map[string]string | Group-by dimension values |
| Count | int | Number of events in this group |
| TotalCost | float64 | Total cost (cost rollup) |
| SuccessRate | float64 | Fraction of successful agents, 0‚Äì1 (success_rate rollup) |
| AvgRetries | float64 | Average retry count per agent (retry_count rollup) |

### Available rollup functions

| Function | Rollup type | Source events |
|---|---|---|
| `ComputeCostRollup` | `"cost"` | `CostEvent` |
| `ComputeSuccessRateRollup` | `"success_rate"` | `AgentPerformanceEvent` |
| `ComputeRetryRollup` | `"retry_count"` | `AgentPerformanceEvent` |
| `ComputeTrend` | any of the above | Time-bucketed trend; returns `TrendResult` |

`Store.GetRollup` delegates to these three functions.

### ComputeTrend

Divides a time range into N equal-width buckets and computes the metric per bucket.

Signature: `ComputeTrend(ctx context.Context, opts ComputeTrendOpts) result.Result[TrendResult]`

| `ComputeTrendOpts` field | Type | Description |
|---|---|---|
| `Store` | `Store` | Observability store to query (required) |
| `Metric` | string | `"cost"`, `"success_rate"`, or `"retry_count"` |
| `TimeRange` | time.Duration | Length of the full window (e.g. `7 * 24 * time.Hour`) |
| `Buckets` | int | Number of time buckets; defaults to 1 if ‚â§ 0 |

Returns `TrendResult.Buckets` ‚Äî each `TrendBucket` has `Start`, `End`, `Value` (the
metric), and `Count` (number of events in that bucket).

## Query helper functions

These functions in `pkg/observability/query.go` wrap `Store.QueryEvents` with
domain-specific logic.

| Function | Returns | Description |
|---|---|---|
| `Query(ctx, store, filters)` | `result.Result[QueryData]` | Generic event query; returns empty slice (not error) when no matches |
| `GetAgentHistory(ctx, store, agentID, limit)` | `[]AgentPerformanceEvent` | Recent performance events for one agent, newest first |
| `GetIMPLMetrics(ctx, store, implSlug)` | `*IMPLMetrics` | Aggregated cost, duration, success rate, agents failed, waves completed |
| `GetProgramSummary(ctx, store, programSlug)` | `*ProgramSummary` | Cost, IMPL count, duration, overall success rate across a program |
| `GetCostBreakdown(ctx, store, implSlug)` | `map[string]float64` | Per-agent cost totals for an IMPL |
| `GetFailurePatterns(ctx, store, filters)` | `[]FailurePattern` | Failure types ranked by occurrence count, with affected agents |

### IMPLMetrics fields

| Field | Type | Description |
|---|---|---|
| TotalCost | float64 | Sum of all CostEvent.CostUSD for the IMPL |
| TotalDurationMin | float64 | Sum of agent durations in minutes |
| SuccessRate | float64 | Fraction of performance events with status `"success"` |
| AgentsFailed | int | Count of performance events with status `"failed"` |
| WavesCompleted | int | Count of `wave_merge` activity events |

### ProgramSummary fields

| Field | Type | Description |
|---|---|---|
| TotalCost | float64 | Sum of all CostEvent.CostUSD for the program |
| IMPLCount | int | Distinct IMPL slugs seen in performance events |
| TotalDurationMin | float64 | Sum of agent durations in minutes |
| OverallSuccessRate | float64 | Fraction of performance events with status `"success"` |

### FailurePattern fields

| Field | Type | Description |
|---|---|---|
| FailureType | string | One of the E19 failure type values |
| Count | int | Total occurrences |
| AffectedAgents | []string | Distinct agent IDs that reported this failure type (sorted) |

Results are sorted by `Count` descending.

## polywave-tools query command

```
polywave-tools query events [flags]
```

### Flags

| Flag | Default | Description |
|---|---|---|
| `--type` | (all) | Event type filter: `cost`, `agent_performance`, `activity` |
| `--impl` | (all) | IMPL slug filter |
| `--program` | (all) | Program slug filter |
| `--agent` | (all) | Agent ID filter |
| `--since` | (none) | Time range: `24h`, `7d`, `30d`, or any Go duration string |
| `--limit` | `100` | Maximum results |
| `--format` | `table` | Output format: `table`, `json`, `csv` |
| `--store` | `~/.polywave/observability.db` | SQLite file path or DSN |

The `--since` flag accepts day suffixes (`7d`, `30d`) in addition to standard Go
duration strings (`24h`, `1h30m`).

### Output formats

**table** ‚Äî tab-separated columns: `ID` (first 12 chars), `TYPE`, `TIMESTAMP`
(RFC3339), `DETAILS` (type-specific one-liner).

**json** ‚Äî full JSON array of all matched events, indented.

**csv** ‚Äî headers `id,type,timestamp,details`; `details` matches the table format.

### Examples

```sh
# All events for an IMPL in the last 7 days
polywave-tools query events --impl add-auth --since 7d

# Only cost events, JSON output
polywave-tools query events --type cost --format json

# Performance events for a specific agent, last 24 hours
polywave-tools query events --type agent_performance --agent A --since 24h

# All events from a specific store file
polywave-tools query events --store /path/to/custom.db --limit 500
```

## polywave-tools metrics command

```
polywave-tools metrics <impl-slug> [flags]
```

| Flag | Description |
|---|---|
| `--program <slug>` | Show program-level summary instead of IMPL metrics |
| `--breakdown` | Show per-agent cost breakdown below IMPL metrics |
| `--store <path>` | SQLite file path (default: `~/.polywave/observability.db`) |

Outputs a tab-formatted table. With `--breakdown`, a second table lists cost by agent.

## Emitter

`*Emitter` wraps `Store` with two write methods.

| Method | Behavior |
|---|---|
| `Emit(ctx, event)` | Writes asynchronously in a goroutine. Errors go to stderr. Never blocks the caller. |
| `EmitSync(ctx, event)` | Writes synchronously. Returns `result.Result[EmitData]` with the event ID and timestamp on success. |

A nil `*Emitter` is safe to call ‚Äî both methods return immediately. `NewEmitter(nil)`
returns nil.

`EmitData` fields: `EventID string`, `Timestamp time.Time`.
